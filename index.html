AI Agent Systems I Built

Self-initiated, then adopted as team standard. Each system pairs an LLM orchestration layer with real tool-use against internal data, and is governed by an explicit evaluation harness. Framed below in transferable terms — the patterns move to any domain.

3-Agent Triage & Root-Cause Orchestrator
Designed, built, and operated · in daily production
Multi-Agent Orchestration
Problem
A 10-person team spent ~20 minutes per case manually reconstructing why an automated decision fired — hopping between chat, the data lake, and a case tracker by hand. High volume, repetitive, judgment-light.
Approach
A 3-agent pipeline orchestrated as a 9-state workflow machine: an intent classifier scopes whether a request is automatable, an investigator agent queries the data lake via async tool-use to reconstruct the decision trace, and a synthesizer writes a plain-language explanation back to the team and updates the case tracker. Scheduled and retried via Airflow.
Feedback
Recurring root-cause patterns are fed back into model monitoring and recalibration — the agent system doubles as an always-on data-quality sensor.

02 — Rigor

How I Keep Agents Trustworthy

Shipping an agent is the easy half. The hard half is proving it stays correct as prompts, models, and data drift. Every system above is governed by an explicit evaluation harness.

Evaluation Framework

Golden sets + LLM-as-judge, scored on four axes

  • Faithfulness — does the output stay grounded in the retrieved evidence, with no invented facts?
  • Hallucination rate — explicitly measured; critical with long-context and document inputs.
  • Completeness — staged checks that the agent covered every required step, not just the easy ones.
  • Human-agreement — agent decisions benchmarked against expert reviewers as the ground-truth ceiling.

Production Discipline

What turns a demo into something a team relies on

  • Human-in-the-loop routing — agents handle clear-cut cases and escalate ambiguity instead of guessing.
  • Orchestration over monoliths — specialized agents with narrow jobs, composed in a state machine.
  • Closed feedback loop — recurring patterns flow back into model monitoring & recalibration.
  • Scheduled & retried — Airflow-orchestrated pipelines with real operational handling, not one-off scripts.
